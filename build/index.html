Fixing the "No 'Access-Control-Allow-Origin' header" error

‚ö†Ô∏è The Problem

Your frontend at https://coinwavezfrontend.netlify.app is being blocked from accessing the backend at https://coinwavezbackend.netlify.app due to CORS policy.

// Browser console error:
Access to fetch at 'https://coinwavezbackend.netlify.app/api/news?filter=rising'
from origin 'https://coinwavezfrontend.netlify.app' has been blocked by CORS policy:
No 'Access-Control-Allow-Origin' header is present on the requested resource.

This happens because browsers enforce the same-origin policy for security reasons, blocking requests from different domains unless the server explicitly allows it.

‚úÖ The Solution

You need to configure your backend to include the proper CORS headers to allow requests from your frontend domain.

// Enhanced CORS configuration for Express.js
const corsOptions = {
  origin: [
    'https://coinwavezfrontend.netlify.app',
    'http://localhost:3000',
    'http://localhost:3001'
  ],
  methods: ['GET', 'POST', 'PUT', 'DELETE', 'OPTIONS'],
  allowedHeaders: ['Content-Type', 'Authorization', 'X-Requested-With'],
  credentials: true,
  optionsSuccessStatus: 200
};

// Apply CORS middleware
app.use(cors(corsOptions));

// Handle preflight requests
app.options('*', cors(corsOptions));

Additionally, you should add specific CORS headers to your news endpoint to ensure it works correctly.
